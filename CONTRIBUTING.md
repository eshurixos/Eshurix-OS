# Contributing to Eshurix OS

## Purpose

This document defines the contribution process for Eshurix OS.

## Development Principles

- Blueprint First
- Documentation Before Implementation
- Small, focused commits
- Follow coding standards
- Follow file standards
- Maintain backward compatibility
- Keep architecture modular
- Prioritize security and scalability

## Pull Requests

Every Pull Request should:

- Be linked to an approved task
- Pass all validation checks
- Include documentation updates if required
- Keep changes focused
- Avoid unrelated modifications

## Commit Messages

Examples:

docs: update architecture documentation

feat: add configuration service

fix: resolve authentication bug

refactor: improve dependency injection

chore: update tooling

## Code Review

Every contribution should be reviewed before merging into the main branch.

## Security

Never commit:

- Secrets
- Passwords
- API Keys
- Tokens
- Private certificates
