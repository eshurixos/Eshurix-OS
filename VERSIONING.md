# Versioning Strategy

## Version Format

Eshurix OS follows Semantic Versioning:

MAJOR.MINOR.PATCH

Example:

1.0.0

## Rules

MAJOR:
Breaking architectural changes.

MINOR:
New backward-compatible features.

PATCH:
Bug fixes, documentation updates, and minor improvements.

## Release Types

- Alpha
- Beta
- Release Candidate (RC)
- Stable

## Principles

- Every release must be tagged.
- Every release must have release notes.
- Production releases require testing and review.
- Breaking changes must be documented.
